# Layout Lane Onboarding Guide

**Author:** Rebecca (Layout Lane)
**Last Updated:** 2026-03-23

## Purpose
This guide helps new contributors understand Layout Lane boundaries, update rules, and review process.

---

## File Boundaries
- Only edit frontend/web/components/layout.php and frontend/web/public/assets/app.css for global/layout changes
- Do not edit frontend/web/views/*.php (template lane)
- Do not edit docs/PLANNING_DOCS/STAGE2_DOCS/ verification evidence (QA/Docs lane)

## Update Rules
- Propose new utility classes in component_catalog.md before using
- Document all new CSS variables and tokens in DESIGN_TOKENS_AND_SPACING.md
- Use comments in CSS/PHP for all major changes

## Review Process
- All changes are reviewed by both Layout and Template Lane owners
- Major changes require a summary in WIREFRAME_STYLE_IMPLEMENTATION.md
- Coordinate with Template Lane for any shared class needs

---

## Notes
- For questions, add to Known Gaps & Open Questions in WIREFRAME_STYLE_IMPLEMENTATION.md
- See component_catalog.md and visual_style_reference.md for reference
