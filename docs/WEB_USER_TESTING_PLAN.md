# Web User Testing and Iteration Plan

## Goal
Run lightweight usability testing with 5 to 10 users focused on first-run onboarding and map/alert comprehension.

## Participant Profile
- 2 to 4 first-time users
- 2 to 4 returning users
- Optional: 1 to 2 users who travel frequently

## Test Scenarios
1. Start as guest and find active alerts.
2. Register and complete onboarding.
3. Use map layers to locate high-severity risks.
4. Ask assistant for context and interpretation.
5. Configure one travel trip rule and delivery options.

## Metrics to Capture
- Time to complete each task.
- Error points and confusion moments.
- First-click success rate on key actions.
- SUS-style rating (1 to 5) after session.

## Session Script (15 to 20 min)
- 2 min intro
- 10 to 14 min task flow
- 3 to 4 min debrief

## Iteration Loop
1. Group findings by severity (critical, high, medium, low).
2. Ship top 3 UX fixes.
3. Re-run mini validation with 2 to 3 follow-up users.
4. Update `docs/TODO.md` and evidence index with outcomes.
