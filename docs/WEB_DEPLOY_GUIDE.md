# Web Deploy Guide (Reproducible Pipeline)

## Deployment Targets
This guide documents deploy-ready workflow patterns for:
- Backend API: Heroku or AWS App Runner/EC2
- Web frontend (PHP): Netlify (static export fallback) or traditional PHP host
- Database: MySQL/MariaDB with migration/backfill steps

## 1. CI Pipeline Stages
Use the workflow in `.github/workflows/web-ci.yml`:
1. Install Node and Python dependencies
2. Build web assets (`npm run build:web`)
3. Run backend verification (`npm run backend:check`)
4. Run web E2E and accessibility checks (`npm run web:test:e2e`, `npm run web:test:a11y`)

## 2. Required Secrets
Set in GitHub Actions secrets and hosting providers:
- `JWT_SECRET_KEY`
- `EMAIL_ENCRYPTION_KEY`
- `DATABASE_URL`
- `LLM_API_KEY` or `OPENROUTER_API_KEY`
- `FIRECRAWL_API_KEY` (if scraper path used)

## 3. Backend Deploy (Heroku Example)
1. Provision app and add MySQL add-on or external DB.
2. Set config vars listed above.
3. Deploy branch.
4. Run migration/bootstrap command:

```bash
python backend/db/init_db.py
python backend/scripts/backfill_summary_alert_links.py
python backend/scripts/migrate_emails_to_encrypted.py
```

5. Validate with:

```bash
python backend/scripts/verify_normalization_guardrails.py
```

## 4. Backend Deploy (AWS Example)
1. Build container image for FastAPI service.
2. Push to ECR and deploy to App Runner/ECS.
3. Inject secrets through AWS Secrets Manager/SSM.
4. Run same migration/backfill commands as above in one-off task.

## 5. Frontend Deploy
### Option A: PHP host
- Deploy `frontend/web/public`, `frontend/web/views`, `frontend/web/services`, `frontend/web/components`, and `frontend/web/config`.
- Ensure `frontend/web/public/assets` is generated by `npm run build:web` before release.

### Option B: Netlify static fallback
- Use pre-rendered static snapshots for evidence/demo pages only.
- Keep dynamic routes on PHP host for full app functionality.

## 6. Post-Deploy Smoke Checks
- `GET /` returns healthy response
- `GET /api/v1/alerts?limit=1` works
- Login/registration flow succeeds
- Assistant loads and sends one message
- Map overlays render

## 7. DB Migration and Rollback Notes
- Migrations and backfills are idempotent; reruns are safe.
- Back up DB before schema-changing release.
- Rollback policy: restore DB snapshot + redeploy previous image.
